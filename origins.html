Origins: How the Cookie Was Invented

The cookie was invented in 1994 by Lou Montulli, who was only 23 years old at the time and working as an engineer at Netscape Communications. Netscape was building an early online shopping system for the company MCI, and the client needed a way to remember a shopper's cart across a bunch of separate page loads.

The problem was that HTTP, the protocol the whole web runs on, does not have any memory built in. Every request a browser makes gets treated like a brand new request, with nothing linking it to the one before it. Without some way to bridge that gap, a shopper would have had to log back in or re enter their cart on every single page.

Some engineers had floated the idea of giving every browser one permanent ID number, but Montulli did not like that idea, since it would let any website track a person everywhere on the web, not just the one site that handed out the ID. So instead he built a system where a server could ask a browser to hold onto a small piece of data and only send it back to that same server later, which kept the memory tied to one site at a time.

He pulled the name from "magic cookies," an older computer science term for a token passed between programs, mostly because he liked how it sounded. Netscape shipped cookie support in October 1994, and within about a year Netscape was the most popular browser in the world, using cookies as an everyday part of browsing. Microsoft added cookie support to Internet Explorer the following year, and from there it spread to basically every browser that came after.

A Quick Timeline

Year What Happened
1994 Lou Montulli invents the cookie at Netscape to solve a shopping cart problem
1994 Netscape ships cookie support in its browser in October
1995 Microsoft adds cookie support to Internet Explorer
1996 to 1997 The IETF works on turning cookies into an open standard
