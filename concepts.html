Key Concepts

Here are six terms that come up again and again in cookie history. Reading through these first can make the other pages on this site easier to follow.

1. HTTP Cookie

A short piece of text, usually under a kilobyte, that a website asks the browser to store and send back on future visits to that same site.

2. Set Cookie Header

The line a server sends in its response to actually create a cookie. The browser reads this header, stores the value, and attaches it to a Cookie header on every later request to that site.

3. First Party Cookie

A cookie created by the site you are actually visiting. These handle logins, shopping carts, language settings, and basic fraud protection.

4. Third Party Cookie

A cookie created by an outside script embedded in the page, most often an advertising network, which lets that company recognize the same browser across many unrelated websites.

5. Cookie Consent Banner

The popup asking permission before a site sets non essential cookies. It exists because of laws like the EU's ePrivacy Directive and GDPR.

6. Privacy Sandbox

Google's set of proposed replacement technologies for third party cookies, meant to support ad targeting without tracking individual people directly.
